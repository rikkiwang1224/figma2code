Figma2Code — 3-SubAgent Architecture

Parent Agent 编排 → design-analyzer → code-generator → visual-validator · 三阶段闭环

Step 1 · design-analyzer-selector

  • • 调用 Figma MCP 解析设计稿节点、样式、层级
  • • 查询 Component Catalog 匹配可用组件
  • • 识别页面模式，生成 componentsTree DSL
  • • 将分析结果写入共享 Disk (.design-analysis.json)

Step 2 · code-generator

  • • 读取 Disk 中的分析结果和样式变量
  • • 批量查询 Component Spec 获取组件 API
  • • 生成多文件 React 代码 (api/adapter/components)
  • • 输出到 output/{conversationId}/ 目录

Step 3 · visual-validator 🆕

  • • Playwright MCP 渲染生成代码，获取运行时截图
  • • 多模态 LLM 比对设计稿 vs 实际渲染
  • • 输出还原度评分 + 差异修复指令
  • • 不达标则反馈 Subagent 2 重新生码（闭环迭代）
